# Архив миссий Токийского магического колледжа

Веб-приложение для централизованного хранения, обновления и формирования отчётов по магическим миссиям. Разработано в рамках лабораторной работы №3.

## Возможности

- Загрузка описаний миссий в форматах JSON, YAML, XML и TXT (два вида форматирования).
- Автоматическая нормализация данных и сохранение в реляционную БД.
- Повторная загрузка с обновлением существующей миссии.
- Частичное редактирование полей через API.
- Просмотр списка всех миссий и полной информации по одной.
- Генерация краткого и детального текстовых отчётов.
- Веб-интерфейс для загрузки и просмотра отчётов.
- Автодокументация API через Swagger (доступна по `/swagger-ui.html`).

## Стек

- **Java 17+**, Spring Boot 3.x
- **Spring Data JPA** (Hibernate)
- **PostgreSQL** (основная БД) / H2 (для локальной разработки)
- **Swagger** (OpenAPI 3)
- **Jackson** (JSON, YAML, XML)
- **Maven**

## Запуск (локально)

1. Запустить PostgreSQL и создать БД `mission_archive`.
2. Настроить подключение в `src/main/resources/application.properties`:
   ```properties
   spring.datasource.url=jdbc:postgresql://localhost:5432/mission_archive
   spring.datasource.username=postgres
   spring.datasource.password=твой_пароль
   ```
   Возможно также переключение на H2-профиль: `spring.profiles.active=dev`.
3. Сборка и запуск приложения (выполнить команду в терминале в корне папки проекта):
   ```
   mvn spring-boot:run
   ```
4. После запуска открыть в браузере:
   - Веб-интерфейс: [http://localhost:8080](http://localhost:8080)
   - Swagger-документация: [http://localhost:8080/swagger-ui.html](http://localhost:8080/swagger-ui.html)

## Работа с тестовыми файлами

В папке `mission-files/` лежат образцы миссий для проверки всех парсеров. Загрузить их можно через веб-интерфейс (кнопка "Upload") или напрямую через API эндпоинт:

```
POST /api/missions/upload
Content-Type: multipart/form-data
```

После загрузки отчеты можно получить через кнопки в списке миссий или GET-запросами:
- Краткий: `GET /api/missions/{id}/report/summary`
- Детальный: `GET /api/missions/{id}/report/detailed`

## Структура проекта (ключевые пакеты)

- `model` – внутренняя модель данных (не зависит от БД)
- `entity` – JPA-сущности
- `dto` – объекты для передачи по API
- `parser` – парсеры разных форматов
- `service` – бизнес-логика загрузки и отчетов
- `report` – генераторы текстовых отчетов
- `controller` – REST-контроллеры
